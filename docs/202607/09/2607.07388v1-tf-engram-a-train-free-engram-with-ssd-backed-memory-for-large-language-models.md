## 摘要
大型语言模型（LLMs）将事实知识和领域特定模式隐式存储在密集的Transformer参数中，导致通过预训练、微调、检索增强或更长的上下文进行知识扩展成本高昂。Engram式记忆提供了一条紧凑的隐藏状态注入路径，但现有的GPU驻留设计通常依赖基于哈希的压缩，导致不相关的短语在共享槽位中冲突，削弱了短语级别的语义保真度。我们提出TF-Engram，一种无训练的Engram系统，它离线从外部语料库构建短语特定的语义记忆，在GPU-DRAM-SSD层级中存储大型记忆表，并使用早期退出引导预测预取来隐藏自回归解码过程中的外部内存延迟。在Qwen3-0.6B上，TF-Engram将平均下游分数从57.6提升至59.4，优于冻结主干网络和参数匹配的LoRA基线。系统评估表明，大型TF-Engram表可以以适度的离线成本构建，基于SSD的存储显著降低了GPU内存需求，预测预取恢复了许多由外部内存访问引起的吞吐量损失。这些结果证明了静态短语记忆可以作为可扩展、无训练且低开销的系统组件集成到LLM推理中。

## Abstract
Large Language Models (LLMs) store factual knowledge and domain-specific patterns implicitly in dense Transformer parameters, making knowledge expansion costly through pretraining, fine-tuning, retrieval augmentation, or longer contexts. Engram-style memory offers a compact hidden-state injection pathway, but existing GPU-resident designs often rely on hash-based compression, causing unrelated phrases to collide in shared slots and weakening phrase-level semantic fidelity. We present TF-Engram, a train-free Engram system that constructs phrase-specific semantic memory offline from external corpora, stores large memory tables across a GPU--DRAM--SSD hierarchy, and uses Early-Exit Guided Predictive Prefetching to hide external-memory latency during autoregressive decoding. On Qwen3-0.6B, TF-Engram improves the average downstream score from 57.6 to 59.4, outperforming both the frozen backbone and a parameter-matched LoRA baseline. System evaluation shows that large TF-Engram tables can be built with moderate offline cost, SSD-backed storage substantially reduces GPU memory demand, and predictive prefetching recovers much of the throughput loss caused by external memory access. These results demonstrate that static phrase memory can be integrated into LLM inference as a scalable, train-free, and low-overhead system component.

## 1. 论文的核心问题与整体含义

- **研究动机**：大型语言模型（LLMs）将事实知识和领域模式隐式存储在密集的Transformer参数中，导致知识扩展成本高昂——预训练、微调、检索增强或扩展上下文均需大量资源。现有Engram式记忆虽提供紧凑的隐藏状态注入路径，但存在三个关键问题：
  1. 记忆构建依赖训练且与模型绑定，无法跨模型复用；

- **整体含义**：提出TF-Engram，将Engram从一个小型哈希压缩的学习表转变为存储管理的无训练外部记忆系统，实现可扩展、低开销的静态短语记忆集成。

## 2. 方法论

### 2.1 核心思想
- **无训练离线构建**：从语料库挖掘候选短语（n-gram、命名实体等），使用冻结的语义编码器（Qwen3-Embedding）生成短语嵌入，构建短语ID→语义向量的记忆表。
- **SSD层次存储**：将记忆表组织为三层：GPU缓存（热数据）、主机DRAM（温数据）、NVMe SSD（冷数据）。保持短语级逻辑条目，避免哈希冲突，容量可独立扩展。
- **Early-Exit引导预测预取**：在LLM中间层（如第L-r层）附加辅助预取头，生成近似下一个token分布，预测未来可能访问的短语，在剩余层计算期间异步预取SSD/DRAM中的数据，掩盖外存延迟。

### 2.2 关键技术细节
- **记忆构建**：
  - 短语挖掘：从FineWeb-Edu、Wikipedia、C4英文、arXiv、PubMed等语料中提取高频n-gram。
  - 语义编码：使用冻结的Qwen3-Embedding模型生成1024维FP16向量。
  - 两级索引：token前缀→短语ID→存储地址，支持最长匹配策略。
- **记忆注入**：将检索到的记忆向量通过门控残差注入到LLM隐藏状态：`h' = h + g·m`，门控g由当前隐藏状态和记忆向量共同决定。
- **预取机制**：
  - 早期退出头在中间层输出近似分布`p_early(x_{t+1}|x<=t)`，选取Top-K候选token。
  - 通过token-to-phrase索引映射为候选短语，优先级分数`s_i = p_i * r_i * c_i`（预测概率、短语有用性、存储层级收益）。
  - 预取在剩余层计算期间异步进行，隐藏窗口大小约为`T_hide ≈ T_compute(剩余层)`。

### 2.3 公式与算法流程（文字说明）
1. 离线：语料 → 短语挖掘 → token化 → 语义编码 → 构建索引和分层存储。
