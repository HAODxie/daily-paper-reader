## 摘要
在线社交媒体帖子为早期抑郁症筛查提供了可扩展的信号，近期研究主要通过风险帖子选择、症状锚定和临床信息特征构建来改进分类前证据。然而，这些筛查阶段的设计通常将最终决策留给单一检测器，忽略了用户在筛查后如何以异质性方式表达抑郁风险。单一分类器必须在异质用户之间进行平均，这可能会稀释局部证据并导致误分类，尤其是对于非自我表露的用户。为了解决这个问题，我们提出了WPG-MoE，一个基于共享大型语言模型（LLM）主干的弱先验引导的密集混合专家框架。WPG-MoE推导出用户级弱语义先验，以软性地将用户路由到匹配不同证据布局的专家。我们将这个过程形式化为使用特权信息的学习（LUPI）：丰富的LLM提取的结构化证据指导训练时路由，而推理时仅保留患者健康问卷-9（PHQ-9）模板筛查和可部署的主干。在中文和英文数据集上的实验表明，WPG-MoE以可解释的路由行为优于强基线。

## Abstract
Online social media posts provide scalable signals for early depression screening, and recent studies mainly improve pre-classification evidence through risk-post selection, symptom grounding, and clinically informed feature construction. However, these screening-stage designs often leave final decisions to a single detector, overlooking how users heterogeneously express depressive risk after screening. A monolithic classifier must average across heterogeneous users, which may dilute localized evidence and cause misclassification, especially for non-self-disclosing users. To address this issue, we propose WPG-MoE, a weak-prior-guided dense mixture-of-experts framework built on a shared large language model (LLM) backbone. WPG-MoE derives user-level weak semantic priors to softly route users to experts matched to different evidence layouts. We formulate this process as learning using privileged information (LUPI): rich LLM-extracted structured evidence guides training-time routing, while inference retains only Patient Health Questionnaire-9 (PHQ-9) template screening and the deployable backbone. Experiments on Chinese and English datasets show that WPG-MoE outperforms strong baselines with interpretable routing behavior.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **问题**：现有社交媒体抑郁检测方法在风险筛查后，使用单一分类器对所有用户做出最终判断。然而，用户表达抑郁风险的方式高度异质：有的直接自我披露（如诊断、用药），有的通过跨越数周的症状片段，有的仅靠少数高强度的孤立帖子。
- **后果**：单一分类器被迫平均这些不同模式，导致局部证据（尤其是稀疏或非自我披露的线索）被稀释，造成误分类。
- **背景**：已有工作改进证据检索（如PHQ-9导向、LLM辅助标注），但最终分类器仍然是“一个模型对付所有用户”，忽视了筛选后的证据布局差异。
- **动机**：需要一种能够根据用户实际证据模式进行条件性专化的检测框架，而不是硬性划分亚型。

## 2. 论文提出的方法论

### 核心思想
- 提出 **WPG-MoE**（Weak-Prior-Guided Dense Mixture-of-Experts），在共享LLM主干上构建密集混合专家系统。
- 利用“弱先验”（用户级别的三种证据倾向：自我披露、片段支持、稀疏高风险）软性地将用户路由到不同专家，而非硬聚类。
- 采用 **LUPI（Learning Using Privileged Information）** 范式：训练时利用外部LLM提取的结构化证据（Path A）指导路由和弱先验构建，推理时完全移除这些特权信息，仅保留PHQ-9模板筛查（Path B）和共享主干。

### 关键技术细节
1. **双路径证据构建**
   - **Path A（训练特权）**：用Qwen3-max对每个帖子提取6类属性（症状、危机、锚点、持续时间、置信度、自我证据），合成帖子风险分数，进一步构建证据块和弱先验。
   - **Path B（可部署推理）**：基于PHQ-9症状模板的余弦相似度筛选风险帖子，动态选择K个候选（预算率12.5%）。
   - 训练时通过4种随机扰动（风险源交换、元数据丢弃、块丢弃、先验丢弃）对齐双路径，避免模型依赖特权信息。

2. **弱先导用户建模**
   - 从Path A结果推导三种弱先验概率：π_self、π_epis、π_sparse。
   - 历史帖子分成8个时序片段，构建全局历史表征H。
   - 构建5个视图（3个倾向专化、1个混合、1个全局残差），通过注意力和池化得到z_sd, z_ep, z_sp, z_mix, z_g。
   - 5个专家各自接收对应视图+元数据，通过门控网络（基于所有视图+弱先验+元数据）输出软权重，加权融合专家输出得到用户状态h。

3. **损失函数**
   - L = L_cls + α L_route + β L_evi + γ L_bal + δ_t L_ent
   - L_route：对齐前三个专家门权重与置信的弱先验（仅当最大先验≥0.6且超过第二大先验≥0.1时使用）。
   - L_ent：余弦衰减的熵正则项，鼓励早期路由多样性。

4. **推理**
   - 移除Path A、证据块、弱先验、危机标注；仅使用PHQ-9模板筛选帖子、共享主干、8段历史摘要和用户统计。

## 6. 论文的主要结论与发现

1. **切片异质性存在**：自披露和片段支持用户容易检测，混合/其他和稀疏用户困难，多种骨干模型上一致。
2. **WPG-MoE显著优于所有基线**：在所有9个跨数据集单元格的F1、AUPRC上多数取得最优；AUROC上有少数被E2-LPS或DeCapsNet匹敌，但整体领先。
3. **路由行为可解释**：训练时自披露专家的门权重最高，测试时门权重根据目标域自适应调整（例如从Twitter到eRisk时片段专家权重上升）。
4. **消融关键组件**：移除MoE导致最大性能下降（6-10%），其次是移除路径A或双路径扰动；弱先验和路由损失贡献较小但稳定。
5. **案例研究**：稀疏证据用户被扁平模型忽略，但WPG-MoE通过激活稀疏/混合/全局专家成功检测；片段支持用户被WPG-MoE的片段+全局专家捕捉。
6. **缺乏强健的证据类型硬分类**：受控混合显示增加其他切片阳性用户不会损害目标切片性能，说明证据布局是兼容倾向而非孤立类别，支持密集MoE软路由。
