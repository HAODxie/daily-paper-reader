## 摘要
在线脑-机接口研究需要能够采集多模态生理数据、训练和更新解码器、运行实时推理，并在可重现的工作流程中保留完整实验溯源的软件。我们提出了Dendrite，一款用Python编写的实时脑-机接口应用，它将信号采集、解码器训练和实时推理整合在一个即开即用且可修改的应用中。Dendrite同时记录多个信号流，每个流以其原生速率记录，并针对它们并发执行多种处理模式。解码器可以从先前训练的模型开始，或者在管道持续运行时在会话中期进行拟合，相同的记录也可用于同一应用中的离线训练。每一次记录、解码器和训练运行都在数据库中追踪，每个解码器记录其配置以及用于训练它的源记录，因此部署的解码器可以追溯到其产生来源。实验范式保持外部化，是一个用任何语言编写的独立程序，通过网络与Dendrite通信，而不是内置在运行时中的模块。我们在内部和公开的BCI数据集上对整个系统进行了端到端验证，在管道实时运行时在线训练和更新解码器。Dendrite在GPL-3.0许可证下开源，地址为https://github.com/dendrite-bci/dendrite。其成果是一个用于开发和评估在线BCI范式的可重现、开源生物医学计算系统。

## Abstract
Online brain-computer interface research requires software that can acquire multimodal physiological data, train and update decoders, run live inference, and preserve the full experimental provenance in a reproducible workflow. We present Dendrite, a real-time brain-computer interface application in Python that brings signal acquisition, decoder training, and live inference together in a single, ready-to-run application that stays modifiable. Dendrite records several signal streams at once, each at its native rate, and executes multiple processing modes concurrently against them. A decoder can start from a previously trained model or be fit mid-session while the pipeline keeps running, and the same recordings feed offline training in the same application. Each recording, decoder, and training run is tracked in a database, and every decoder records the configuration and the source recordings it was trained from, so a deployed decoder traces back to what produced it. The experimental paradigm stays external, an independent program in any language that reaches Dendrite over the network, rather than a module built inside the runtime. We validate the full system end-to-end on in-house and public BCI datasets, training and updating decoders online while the pipeline runs in real time. Dendrite is open-source under the GPL-3.0 license at https://github.com/dendrite-bci/dendrite. The result is a reproducible, open-source biomedical-computing system for developing and evaluating online BCI paradigms.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：在线脑-机接口研究需同时完成多模态生理信号采集、解码器训练与更新、实时推理，并保证实验流程的可重复性。现有软件要么功能碎片化（采集、训练、分析工具分离），要么将实验范式紧密绑定在系统内部，难以灵活修改和溯源。
- **背景**：BCI研究是迭代的：设计范式→采集数据→离线训练解码器→部署在线推理→修正范式。这一循环要求软件既支持实时多流采集、动态训练/更新解码器，又允许外部实验范式独立演化。现有平台（如BCI2000、OpenViBE）功能完整但封闭；Python 科学栈（如MNE、scikit-learn）是离线优先，缺乏实时运行时。
- **整体含义**：Dendrite 旨在提供一个**即开即用、可修改的实时Python BCI应用**，统一采集、处理、训练、推理、回放和数据溯源，同时保持实验范式外置（通过LSL事件协议），降低软件负担，提升可重复性。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：将系统分为四个层次（数据层、处理层、机器学习层、Web界面层）加一个外置任务应用协议（LSL事件）。各层间窄接口，硬件无关化，并发模式共享状态而非固定管道图。
- **关键技术细节**：
  - **数据层**：通过Lab Streaming Layer (LSL) 实现硬件无关的流发现与传输；每个流独享零拷贝共享内存环形缓冲区（多读者单写入者，SPMC），避免慢消费者阻塞采集；HDF5+SWMR 实时写入，SQLite 记录实验谱系（研究、记录、解码器、训练任务）。
  - **处理层**：三个处理模式各自运行在独立子进程中，读取同一环形缓冲区：
    - **同步模式**：事件触发分窗，每N个epoch触发重新训练，在线预训练精度（prequential accuracy）跟踪。
    - **异步模式**：连续滑动窗口分类，可从同步模式热切换新解码器。
    - **神经反馈模式**：频带功率估计（4Hz更新率）。
  - **机器学习层**：统一scikit-learn fit/predict接口，支持神经解码器（EEGNet，PyTorch后端）与经典管道（CSP+LDA, xDAWN等）；串行化后可直接加载到实时模式；训练与推理分离子进程，不阻塞管道。
  - **实验范式外置**：通过LSL事件流（JSON格式）与预测流（支持LSL、ROS2、TCP/UDP、ZeroMQ）交互，范式可用任意语言编写独立程序。
  - **Web界面**：FastAPI + Vue 3 单页应用，提供控制面板、数据浏览器、ML工作台；通过四个WebSocket通道（遥测、可视化、模式数据、训练）实时更新。
- **算法流程（文字说明）**：
  1. 配置采集流（EEG, EMG, 事件等）并启动Dendrite。
  2. 同步模式捕获事件预定标记，提取epoch；每10个epoch发送训练请求给ML服务（从SWMR文件读取epoch数据）。
  3. ML服务在子进程中训练解码器（支持Optuna超参数搜索），完成后将解码器发布到共享状态。
  4. 同步模式加载新解码器用于后续试验；异步模式检测到共享状态更新后热切换，连续分类预测。
  5. 结果通过输出流发送给外部应用（如刺激程序）。
  6. 所有记录、解码器、训练运行存储在HDF5与SQLite中，可追溯。

## 6. 主要结论与发现

- Dendrite能够端到端运行在线BCI流程：采集、实时处理、在线训练（每10个epoch重新拟合）和热切换解码器，且训练不中断推理输出。
- 在三个基准数据集上，在线训练的解码器精度均高于机会水平（内部MI平均κ=0.36，4类MI平均κ=0.31，P300平均κ=0.46）。
- **实时性能满足在线BCI约束**：
  - 异步模式步进间隔100ms（或～97.7ms），单步推理延迟1-4ms（p50），远小于步进间隔。
  - 端到端闭环延迟中位数约13ms，p99约66ms，主导因素是网络传输而非计算。
  - 在线重新训练子进程独立，不干扰推理节奏。
- 系统支持多模态流同时采集（EEG、EMG、运动学），每个流以原生速率记录，通过共享内存缓冲区保持同步。
- 通过LSL事件协议，外部范式可以独立用任何语言实现，并实时接收预测结果。

## 7. 优点

- **全面集成**：在一个可运行的Python应用中整合了采集、处理、训练、推理、回放、监控和数据溯源，无需用户自行组装工具链。
- **模块化与并发**：处理模式作为独立子进程，崩溃隔离；共享环形缓冲区允许多种模式并发读取相同数据，且训练和推理分离。
- **实验范式外置**：通过LSL事件而非内嵌模块，保持范式灵活性，支持任何语言编写的刺激程序。
- **数据溯源**：每个记录、解码器、训练任务都在SQLite中记录来源（配置、源记录ID、精度等），部署的解码器可完全追溯，提升可重复性。
- **低延迟**：内部计算延迟毫秒级，端到端闭环延迟在p99 66ms，满足在线BCI需求。
- **开源与可修改**：全部代码开源（GPL-3.0），研究者可在Python层面修改任何部分。
