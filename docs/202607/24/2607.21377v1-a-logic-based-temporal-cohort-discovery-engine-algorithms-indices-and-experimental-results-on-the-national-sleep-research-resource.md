## 摘要
大型睡眠研究数据库包含丰富的时间戳生理注释，但队列发现通常仍通过临时脚本或标量索引过滤器实现。我们提出了一种基于逻辑的时间性队列发现引擎，将形式语义、模型检测、专用索引和实证评估统一到生物医学信息学框架中。我们采用理性集成逻辑（QEL）作为睡眠数据查询的稠密时间形式基础，并将每个注释的多导睡眠图表示为生物医学事件结构时间模型（BEST），即从事件标签到不重叠有理区间集合的有限映射。队列发现被形式化为在BEST数据库上对QEL公式进行模型检测。我们将常见的睡眠研究需求组织为三种可重用的时间查询模式：单事件检索、双事件时间模式匹配和事件数据提取。原型队列发现引擎在Python中实现，支持内存和基于MongoDB的执行模式，并在包含多达9000万个区间的合成区间数据集以及来自克利夫兰儿童睡眠与健康研究（CCSHS）的真实世界国家睡眠研究资源注释（包含515名受试者、202,587个区间、23个事件标签）上进行了评估。2DFC以线性空间和线性构建时间构建索引，在9000万个区间上，构建时间从RTFC的11,549秒和2DRT的23,902秒减少到3,655秒。在CCSHS上，队列选择查询在原生规模下以亚秒级延迟执行，在1000倍规模下在45秒内完成。本工作是通讯作者倡导的符号生物医学项目的一部分。

## Abstract
Large sleep-study repositories contain rich time-stamped physiological annotations, but cohort discovery is still commonly implemented as ad hoc scripts or scalar-index filters. We present a logic-based temporal cohort discovery engine that brings formal semantics, model checking, specialized indexing, and empirical evaluation into a unified biomedical informatics framework. We adopt Rational Ensemble Logic (QEL) as a dense-time formal foundation for sleep-data querying and represent each annotated polysomnogram as a Biomedical Event Structure Temporal Model (BEST), a finite mapping from event labels to non-overlapping rational interval ensembles. Cohort discovery is formulated as model checking of QEL formulas over BEST databases. We organize common sleep-research requirements into three reusable temporal query patterns: single-event retrieval, dual-event temporal pattern matching, and event data extraction. The prototype cohort discovery engine was implemented in Python with in-memory and MongoDB-backed execution modes and evaluated on synthetic interval datasets containing up to 90 million intervals and on real-world National Sleep Research Resource annotations from the Cleveland Children's Sleep and Health Study (CCSHS) containing 515 subjects, 202,587 intervals, 23 event labels. 2DFC constructs indexes in linear space and linear build time, reducing build time at 90 million intervals from 11,549 s with RTFC and 23,902 seconds with 2DRT to 3,655 seconds. On CCSHS, cohort-selection queries executed at sub-second latency at native scale and under 45 seconds at 1,000 times scale. This work is a part of the Symbolic Biomedicine program championed by the corresponding author.

### 二、方法论

- **核心思想**：将每个受试者的注释PSG表示为 **BEST（Biomedical Event Structure Temporal Model）**，即从事件标签到有限个非重叠有理区间集合的映射；队列定义用 **QEL（Rational Ensemble Logic）** 公式描述；受试者是否属于队列等价于该BEST是否在指定观测点满足QEL公式（模型检查）。
- **关键概念**：
  - **BEST**：每个事件标签对应一个区间集（interval ensemble），区间不重叠，按起始时间排序。
  - **QEL语法**：包括原子命题（事件标签）、位移操作（\( \varphi_u \)）、未来窗口存在量词（\( \Diamond_t \)）、未来窗口全称量词（\( \Box_t \)）、布尔连接词、一阶量词（∃/∀作用于有理数时间或窗口长度）。
  - **查询模式**：三类可重用模板——单事件检索、双事件时间模式匹配、事件数据提取。双事件模式包括共现、先后关系（四种变体）、有界延迟等。
- **索引技术**：
  - **2DFC（二维分段级联）**：利用BEST中每个标签的区间已排序且不重叠的特点，将每个标签的起始点列表和结束点列表分别构建一维分段级联结构。支持区间重叠查询（单事件检索），查询时间 \( O(\log |FC[0]| + m + k) \)，空间 \( O(n) \)，构建时间 \( O(n) \)。
  - **FCFC（全连接分段级联）**：专为双事件模式匹配设计。将全区间集合的边界点排序为全局索引，为每个事件标签预计算每个全局边界处的“下一个起始”和“下一个未结束”区间位置。模式匹配时从锚定事件边界O(1)定位到目标事件的候选区间，避免重复平铺排序。
- **理论保证**：定理1（有限理性证据集）保证在固定BEST和QEL公式下，密集有理时间上的量化问题只需在有限个端点诱导的单元格中测试代表点即可，从而将模型检查转化为端点数组操作。

### 六、主要结论与发现

1. **形式语义可行**：QEL和BEST提供了数学上清晰且机器可执行的时间队列表达，模型检查可以准确实现复杂临床表型定义（如中央型呼吸暂停后600秒内血氧下降）。
2. **2DFC索引高效**：
   - 构建时间线性 \( O(n) \)，空间 \( O(n) \)。在9000万区间时构建仅需3655秒，较RTFC（11549秒）提升约3倍，较2DRT（23902秒）提升约6倍。
   - 查询时间 \( O(\log n + m + k) \)，在典型睡眠数据集（m≈23）上接近RTFC；当事件类型极少时略慢，当区间极多且事件类型少时RTFC更具优势。
3. **FCFC加速双事件匹配**：在RAM下减少80-98%查询时间，在MongoDB高候选集场景下减少约50%。
4. **真实数据可扩展**：CCSHS原生规模（515受试者）下所有查询亚秒级；1000倍缩放（约50万受试者）下队列选择查询仍可在45秒内完成；事件提取查询因结果集大小导致I/O瓶颈。
5. **模板可复用**：三类查询模板可覆盖大部分睡眠研究常见时序约束（如共现、先后、有界延迟、阶段限制等），并兼容AASM指南定义。

### 七、优点

- **理论扎实**：基于QEL的模型检查具有完备的数学基础（有限证据定理），确保运行时正确性。
- **索引创新**：2DFC利用BEST中区间已排序不重叠的领域特性，实现线性构建和空间，优于通用范围树；FCFC针对模式匹配的O(1)跳转设计。
- **实用性**：框架支持RAM和MongoDB两种存储模式，适合不同规模部署；示例查询直接对接临床表型（如中央呼吸暂停与血氧下降延迟），体现应用价值。
- **可复现性**：QEL公式即配置文件，消除临时脚本造成的歧义。

### 八、不足与局限

- **特性假设**：要求每个事件标签内的区间不重叠。重叠的注解需要预处理或归一化，可能产生近似。
- **空间开销**：FCFC存储 \( O(m l) \)（m为标签数，l为总区间数），当标签多区间多时可能很大，论文建议稀疏或按需构建，但未评估压缩收益。
- **查询范围受限**：仅评估了模板形式的QEL查询；完全通用的嵌套QEL需要递归应用有限证据定理，但未在实验中出现，性能未知。
- **缺乏增量更新**：添加新受试者或新事件需重新构建2DFC/FCFC索引，不适合持续增长的数据流。
- **硬件信息缺失**：未说明实验机器配置，影响结果可复现性。
- **真实数据集局限**：仅使用CCSSH单一队列（515人），未在更大规模或异质性更强的NSRR数据集（数千人）上验证。
- **偏差风险**：合成数据生成方式简单（均匀分布），可能无法反映真实临床事件分布的倾斜性。真实数据的缩放通过复制受试者实现，不能完全模拟独立受试者增长带来的分布多样性。
- **与最新时间序列数据库比较缺失**：未对比如TimescaleDB、InfluxDB等专业时序数据库的查询能力或性能。
